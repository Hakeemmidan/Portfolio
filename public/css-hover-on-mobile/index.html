How to Prevent CSS Hover State From Getting stuck on Mobile Browsers

Problem

The CSS hover state often gets stuck on mobile (touch screen) browsers. It would be something like this:

How to Fix It

Using the ‘hover’ CSS media query, which could be applied either through SCSS or CSS. In the case of SCSS, we’re going to use a mixin.

Here is each solution respectively:

Solution 1: SCSS Mixin

The mixin:

@mixin hover-supported {    
    @media (hover: hover) { 
      @content;
    }
}

Example use:

  .example {
    @include hover-supported() {
      &:hover {
        background-color: black;
      }
    }
  }

Solution 2: CSS

Example use:

  @media (hover: hover) {
    .example:hover {
      background-color: black;
    }
  }

Explanation

Both examples change the background-color of HTML elements with class example only when they’re hovered-over on non-touch screen devices (or any hover supported device). In other words, this applies hover styling on only non-touch screen devices.

PS: I’m not sure how this would act on touch-screen laptops.
